# Ресёрч: стартовые бонусы (`gameRules.bonuses`)

Дата: 2026-06-14. Источники: 82 официальных шаблона, `Core.zip` (`DB/map_bonuses/map_bonuses.json`, `DB/difficulties.json`, Lang), исходники AuroraRMG.

## Как это работает

Запись бонуса в шаблоне:

```json
{ "sid": "add_bonus_res", "receiverSide": -1, "receiverFilter": "start_hero", "parameters": ["gold", "10000"] }
```

- **`sid`** — тип бонуса, ссылка на каталог `DB/map_bonuses/map_bonuses.json` в Core.zip. Каталог канонический: 8 типов, у каждого — теги параметров и значения по умолчанию.
- **`receiverSide`** — чья сторона получает. Во всех официальных шаблонах `-1` («все игроки»); формат позволяет адресовать конкретную сторону, но это не используется.
- **`receiverFilter`** — получатель внутри стороны: `start_hero` (только стартовый герой), `all_heroes` (каждый герой), пусто/отсутствует — бонус стороне, а не герою (для ресурсов и опыта стороны). Допустимые значения перечислены в каталоге (`receiversFilters`).
- **`parameters`** — массив **строк**; структура зависит от типа.

## Каталог типов (из map_bonuses.json)

| sid | тип движка | параметры (теги каталога) | в официальных шаблонах |
|---|---|---|---|
| `add_bonus_res` | sideRes | resource name, value | 5 записей (1 шаблон) |
| `add_bonus_hero_exp` | heroExp | hero exp value | не используется |
| `add_bonus_side_exp` | sideExp | side exp value | не используется |
| `add_bonus_hero_spell` | heroMagicAddition | magic sid | 11 записей (9 шаблонов) |
| `add_bonus_hero_item` | heroItemAddition | item sid | 2 записи |
| `add_bonus_hero_unit` | heroUnitsAddition | unit sid, value | не используется |
| `add_bonus_hero_unit_multipler` *(опечатка игры)* | heroUnitsModifier | множитель | 3 записи (всегда `"2"`) |
| `add_bonus_hero_stat` | heroStat | stat name, value1, value2, value3 | 23 записи |

Наблюдения по данным:

- **Ресурсы**: `gold, wood, ore, gemstones, crystals, mercury, alchemicalDust` (полный список подтверждён по `DB/difficulties.json`).
- **`add_bonus_hero_stat`** в природе встречается в двух видах:
  - `["movementBonus", "0"]` — 15 шаблонов («бонус хода»; значение 0 — видимо, включает стандартный бафф);
  - `["magicCostSidSet", "<spell_sid>", "-999", "0"]` — 8 шаблонов (обнуляет стоимость конкретного заклинания, чаще `neutral_magic_town_portal`).
- **Локализации имён бонусов в игре нет**: токены `bonus_name_N` из каталога нигде в Lang не определены. Подписи в редакторе будут свои.
- **Quirk формата**: в `Wastelands.rmg.json` `bonuses` — **один объект**, не массив. Игра это терпит. Парсер обязан принимать обе формы; экспортировать — всегда массивом.
- Aurora перечисляет лишь 5 sid из 8 и не типизирует параметры — наш каталог полнее.

## План реализации

1. **Модель**: `settings.startingBonuses: StartingBonus[]`, где `StartingBonus = { sid, receiverSide, receiverFilter, parameters: string[] }` — полная запись без потерь.
2. **Импорт**: читать `gameRules.bonuses` (массив или одиночный объект → массив); убрать `bonuses` из passthrough `originalGameRules` (ключ в knownRulesKeys). **Экспорт**: писать массивом, пустой список — опускать.
3. **UI**: секция «Бонусы старта» в левой панели (под «Банами», бонусы общие на карту). Список карточек (подпись типа + резюме параметров + получатель + корзинка) и форма добавления: селект типа → динамические поля:
   - ресурсы: селект из 7 + число;
   - опыт героя/стороны: число;
   - заклинание: пикер из каталога заклинаний (уже есть из банов);
   - артефакт: пикер из артефактов (уже есть);
   - существа: текстовый sid + число (каталога юнитов пока нет);
   - множитель армии: число;
   - характеристика: два пресета — «Бонус хода» и «Бесплатное заклинание» (с пикером заклинания) + режим «вручную» (имя характеристики + до 3 значений), по образцу «Особой» реакции охраны;
   - получатель: «стартовый герой / все герои» для героических типов, скрыт для side-типов (`add_bonus_res`, `add_bonus_side_exp`).
4. **Тосты** добавления/удаления — как в банах.
5. **Тесты**: round-trip всех 8 типов; объектная форма Wastelands; слияние/стабильность канонического экспорта; пустые bonuses не пишутся.
